# DRCDLX — Release Card

Минималистичный веб-интерфейс для представления музыкальных релизов **ТОЛЬКОБИТ** и **АБСОЛЮТНОЕ НИЧЕГО** от артиста **DRCDLX** (2026).  
Создан на HTML + CSS + JS.

---

## 🎯 Цель
- Показать **только ключевую информацию** по релизам: обложка, название, артист, жанр, длительность, год.
- Сохранить визуальную атмосферу: тёмный и минималистичный дизайн.
- Обеспечить доступность и адаптивность — для всех устройств и пользователей.
- Сохранить **минимализм** и **доступность**.

---

## ✨ Особенности
- **Hover-эффекты** для обложки и ключевых слов в метаданных — лёгкое всплытие и тонкая подсветка.
- **Градиент внизу карда** — создаёт ощущение глубины.
- **Индикатор активного плеера** — при фокусе плеер подсвечивается акцентным цветом.
- **Чистая семантика** — доступность, скринридеры, SEO.
- **Адаптивный дизайн** — корректное отображение на мобильных и десктопах.
- **Без лишних элементов** — только обложка, информация и плеер.
- **Кнопка "Скачать"** — с выбором формата (только FLAC).
- **Разделитель между треками** — визуальное разделение.
- **Кнопки "Поделиться"** — копирование ссылки и отправка в ВКонтакте / Одноклассники.
- **Автоматическая остановка** одного трека при воспроизведении другого.
- **Кнопка "Вернуться к началу"** — появляется при прокрутке.
- **Фильтр по типу трека** — возможность отфильтровать треки на сольные и совместные работы.

---

## 🛠 Технологии
- **HTML5** — семантическая разметка (`<article>`, `<audio>`, `<img>`, `<details>`, `<summary>`)
- **CSS3** — переменные, `:hover`, `::after`, `:focus-within`, адаптивные медиа-запросы
- **JavaScript** — копирование ссылки, генерация URL поделки, остановка других треков, фильтрация
- **Шрифт**: Inter (system fallback — Segoe UI, sans-serif)

## 🖼 Особенности дизайна

## 🧠 Как это сделано: технический кейс

### Задача:
Создать минималистичный веб-интерфейс, который позволяет пользователю прослушать и получить информацию о двух треках.

### Решение:
- Использованы только HTML/CSS/JS — без библиотек и фреймворков.
- Автоматическая остановка треков — через `play`-событие и DOM-перебор.
- Фильтрация — через добавление/удаление CSS-класса `hidden`, управляемого JavaScript.
- Доступность — семантика, `:focus-within`, контраст, `alt`, `aria-hidden` где нужно.

### Результат:
Минималистичный, быстрый, доступный интерфейс, который работает на всех устройствах и не требует сервера.
